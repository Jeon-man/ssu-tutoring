### 세계 최초의 전자식 컴퓨터 -> ABC 컴퓨터

### 오늘날의 컴퓨터의 모델인 ‘프로그램 내장형 컴퓨터(stored-program computer)’를 처음 제안한 사람은 폰 노이만 이다.

### 최초의 프로그램 내장 방식의 전자식 컴퓨터는 EDSAC(에드삭) 이다.

### 전기 기계식 계산기와 전기 전자식 계산기(현대의 컴퓨터)의 차이점은? -

- 구동 방식과 내부 소자의 차이

### Turing Test와 CAPTCHA와의 관련성(같은 점, 다른 점)을 기술하라.

- 튜링 테스트는 사람이 기계를 시험하는 것이고, 캡차는 기계가 사람을 시험하는 것이다.

### 내장형 프로그래밍(저장 프로그램)

- 개념: 명령과 데이터를 같은 기억장치에 두고, CPU가 순서대로 읽어 실행.
- 왜 중요: 배선을 바꾸지 않고 기억 내용만 바꿔 프로그램을 바꿀 수 있어 범용 컴퓨터가 됨. **폰 노이만**이 제안함

### 컴퓨터 하드웨어의 기본 구성 요소 4 부분의 이름

- CPU, Memory, 입력장치, 출력장치

### RAM, ROM, Hard Disk, Booting, 이 4개의 단어를 이용하여 Booting의 개념을 설명하는 하나의 문장을 완성하시오.

- 전원이 켜지면 ROM의 BIOS가 실행되고, 하드디스크의 운영체제를 RAM으로 옮겨 쓸 수 있게 만드는 과정이 부팅(Booting)이다.

### 보수·2의 보수 (서술형 핵심만)

- 보수가 필요한 이유: 음수를 뺄셈 없이 덧셈으로 처리해 ALU를 단순화(가산기 중심).
- 2의 보수 구하기: 양수 이진 → 비트 반전(1의 보수) → +1. (또는 오른쪽 첫 1까지 두고 왼쪽만 반전.)
- 2의 보수의 장점: 0이 하나, 덧셈·감산을 같은 회로로 처리, 부호 비트가 크기·오버플로 판단과 잘 맞음.
- n비트 2의 보수 범위: −2<sup>n−1</sup> ~ 2<sup>n−1</sup> − 1 (최소 절댓값이 최대보다 1 큰 이유: 음수 쪽에 값이 하나 더 많음.)
- 부호·절댓값 방식: +0 / −0 두 가지 등으로 낭비·회로 복잡 — 왜 실무 CPU는 2의 보수를 쓰는지와 연결해 말할 수 있으면 됨.

### 부동소수점·지수·히든 비트

- 왜 부동소수점: 가수 + 지수로 스케일을 바꿔, 적은 비트로 큰 수·작은 수를 함께 다루고 동적 범위를 넓힘.
- 초과(excess) 지수: 지수에 바이어스를 더해 저장 → 비교·정렬을 부호 없는 정수처럼 처리하기 쉬움.
- 히든 비트: 정규화 이진 가수의 선행 1을 저장하지 않음 → 정밀도 한 자리 이득 vs 비정규 수 규칙·복원 부담.
- 시험: 비트 채우기(부호·지수·가수 필드에 실수 ↔ 비트열)는 수업에서 준 비트 수·바이어스 그대로 손으로 반복 연습.
- MSB 부호비트로 자르고 오른쪽으로 지수 비트 갯수만큼 자르기, 나머지 가수부

### 4진 (3220)<sub>4</sub> → 십진 (4의 보수로 읽기 = 음수)

4자리 4진을 4의 보수(부호)로 쓸 때는 맨 왼쪽 자리(4³ 자리)가 2 또는 3이면 음수입니다. (3220)<sub>4</sub>는 앞자리가 3이므로 음수로 읽고, 십진 값은 4의 보수 과정으로 크기를 구한 뒤 마이너스를 붙입니다.

1. (3333)<sub>4</sub> − (3220)<sub>4</sub> — 같은 자리끼리만 뺀다 (3의 보수 단계).

```
  3 3 3 3
− 3 2 2 0
---------
  0 1 1 3
```

→ (0113)<sub>4</sub>

2. +1 — (0001)<sub>4</sub> 더해서 4의 보수로 정리.

(0113)<sub>4</sub> + (0001)<sub>4</sub> = (0120)<sub>4</sub>

3. (0120)<sub>4</sub>를 십진으로

0×64 + 1×16 + 2×4 + 0 = 16 + 8 = 24<sub>10</sub> → 이게 절댓값(크기).

4. 부호 붙이기 — 원래 (3220)<sub>4</sub>는 음수였으므로 십진 값은 −24<sub>10</sub>.

### 아날로그 데이터에 비해 디지털 데이터 표현이 가지는 장점과 단점은 무엇일까?

- 장점 - 명확하게 단계별로 수치로 표현할 수 있다.
- 단점 - 데이터 사이사이의 무한대 영역을 세밀하게 표현할 수 없다. 디지털 정보를 담을 비트에 의해 정보의 정밀도가 제한된다

### 컴퓨터 내부에서는 2진수를 표현할 때 부호와 절대값 표현은 사용하지 않고, 2의 보수 표현을 사용한다. 그 이유는 무엇인가?

- 2의 보수 표현을 사용하지 않는다면 부호 비트를 구분하는 절차, 부호비트에 따라 더하기, 빼기 계산 구분해서 수행하여야한다. 하지만 2의 보수 표현을 사용한다면, 부호 비트를 구분하는 절차를 가질 필요가 없으며 피연산자가 음수, 양수라 하더라도 더하기 계산만으로 결과를 구할 수 있다

### 실수(floating-point number) 표현 방식에서 ‘부동 소수점 표현’ 방식과 비교해서 ‘고정 소수점 표현’ 방식의 장점이 무엇이며, 그 장점의 이유는 무엇인가?

- 부동소수점 표현 방식은 부호비트, 지수부, 가수부를 나누어 계산을 하고 합쳐서 실제 수치를 구해야하는 복잡한 연산이 있는 반면 고정소수점 방식은 각자리수에 배정된 2의 배수만 구하면된다.

### 실수 표현을 위해 IEEE 754 floating point 표현 법에서 hidden bit 개념을 사용하는 이유는 무엇인가?

- 정규화로 인해 항상 가수부는 항상 1로 시작한다 그렇기 때문에 1.0을 약속된 수로 상정하고 소수점 1자리를 더 표현하면 표현할 수 있는 범위가 넓어진다

### 진리표 · 논리식 · 논리도

- 시험은 셋 중 하나를 주고 나머지로 변환하는 유형이 흔함.
- 진리표 → 식: 출력이 1인 행을 모아 곱항들의 합(SOP) 등으로 쓰기.
- 식 → 도: AND / OR / NOT으로 단계 나누어 배선.
- 도 → 진리표: 입력 조합마다 출력까지 따라가며 기록. -> 출력부터 게이트 하나씩 타고가기

### 4비트 감가산기 (4-bit adder–subtractor)

4비트 두 수 A, B에 대해 덧셈 또는 뺄셈을 같은 조합 논리 블록으로 처리하는 회로입니다. 뺄셈은 보통 B를 1의 보수로 바꾸고(비트별 XOR 등으로 반전) 최하위 carry in에 1을 넣어 2의 보수 덧셈과 같게 만들거나, 모드 신호 한 줄로 “덧기/빼기”를 전환합니다. 각 비트는 전가산기(FA)로 연결되고, Cout가 옆 단의 Cin으로 이어지는 연쇄 구조입니다.

## 4. 카르노 맵으로 회로 줄이기

- 역할: 진리표(또는 식)을 AND·OR 등으로 만든 회로로 옮길 때, 게이트 수·입력 수를 줄이기 위한 표 위에서의 묶기 도구.
- 맵 쓰는 감각: 같은 출력(보통 1)인 칸을 인접한 덩어리로 크게 묶기(맵 크기에 따라 가장자리가 이어진 것처럼 인접인 경우도 있음). 묶인 덩어리마다 공통으로 남는 입력 조건을 한 항으로 읽어 최종 식으로 만든 뒤 회로도로 그림.
- Don’t care(d, X): 그 입력에서는 출력이 정해지지 않아도 될 때 — 맵에서 1을 더 크게 묶이게 0 또는 1로 잡아도 됨. 그다음 최소 식 → 회로도. (Don't care의 경우 진리표가 주어지거나, don't care의 범위가 주어질 것임)

## 5. 순차 논리 (상태·플립플롭·카운터)

### 상태 비트

- 순차 회로는 지금 상태(플립플롭 출력들)와 입력이 다음 상태를 정함.
- 상태 수가 늘수록 필요한 플립플롭 개수(비트 수)가 늘어남 — 상태 전이표·다음 상태 식 문제와 연결.

### 플립플롭·래치·카운터

- 플립플롭: 클럭 에지에서만 샘플링되어 출력이 바뀌는 1비트 기억. 동기 설계에 유리.
- 래치: 클럭 레벨 동안 입력이 그대로 비칠 수 있음(투명) — 플립플롭과 언제 입력이 반영되는지로 비교 설명.
- 종류: 교재 기준 SR, JK, T 등 이름·특징만 맞추면 됨.
- 카운터: 클럭마다 상태가 순환. 리플(비동기)

### 엣지 트리거(edge-triggered)

- 뜻: 출력(또는 내부 상태)이 바뀌는 시점을 클럭이 레벨로 유지되는 동안이 아니라, 클럭 전이(에지)가 일어나는 짧은 순간에만 입력을 샘플링하는 방식. 플립플롭이 말하는 “클럭 에지에서만”이 이에 해당합니다.
- 상승 에지 트리거 (rising edge, positive-edge triggered): 클럭 0 → 1(저→고)로 바뀌는 순간에만 입력을 반영·상태를 갱신합니다. 회로 기호에 삼각형(에지 표시)만 있거나 “상승”으로 적힌 경우가 많습니다.
- 하강 에지 트리거 (falling edge, negative-edge triggered): 클럭 1 → 0(고→저)로 바뀌는 순간에만 입력을 반영·상태를 갱신합니다. 클럭 입력에 버블(○)이 붙어 있으면 보통 하강 에지에서 동작한다고 보면 됩니다(교재·기호 규칙에 따름).

### 비동기식 2비트 업카운터 (2-bit asynchronous / ripple up counter)

- 역할: 내부 상태가 00 → 01 → 10 → 11 → 00 …으로 돌아가며 0~3을 이진수로 한 단계씩 올리는 카운터입니다. 플립플롭 두 개의 출력이 LSB·MSB입니다.
- 비동기식(리플)의 뜻: 외부 클럭 CLK는 보통 LSB(최하위 비트) FF에만 직접 연결되고, 한 단 위 FF의 클럭은 아래 단 FF의 출력(Q 또는 Q̄ 등, 교재·회로도대로)에서 받습니다. 즉 모든 FF가 같은 CLK 에지에 동시에 바뀌는 것이 아니라, 아래 단이 먼저 바뀐 뒤 전파 지연을 거쳐 위 단이 따라 바뀝니다.
- 엣지와의 관계: 각 FF는 자기 클럭 입력에 들어오는 에지에서만 토글합니다. LSB는 외부 CLK 에지마다 변하고, MSB는 LSB 출력이 자기 클럭 쪽으로 에지를 만들 때 변하므로, 동기식 카운터보다 비트 간 타이밍 차이(리플)가 생길 수 있다는 점이 시험에서 묻는 포인트입니다.
