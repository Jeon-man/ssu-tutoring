## 2. 실수의 정밀도와 오차 문제

비트 수는 유한한데 실수는 무한에 가깝기 때문에 **근사값**을 쓰게 되고, 그 과정에서 **오차**가 생깁니다.

### 실습 예제: 실수 연산의 한계

```c
#include <stdio.h>

void float_test() {
    float a = 0.2;
    float b = 0.2;

    if (a + b == 0.4) {
        printf("정확함\n");
    } else {
        printf("오차 발생\n"); // 실제 실행되는 구간
    }
}
```

**왜 이런 일이 나는가**

- 상수 `0.4`는 보통 **`double`**으로 해석되고, `a + b`는 **`float`** 연산 결과입니다. **비트 수·해석 방식이 다른 두 값**을 `==`로 비교하면 미세한 차이 때문에 다르다고 나올 수 있습니다.

**실무에서의 습관**

- 정밀도가 중요하면 **반복 덧셈**보다 **한 번의 곱셈** 등으로 오차 누적을 줄이는 방향을 고려합니다.

## 4. 부동 소수점의 구조 (IEEE 754)

한 실수를 크게 **세 덩어리**로 나눕니다.

1. **부호 (Sign)** — 양수 `0`, 음수 `1`
2. **지수 (Exponent)** — 전체 **스케일**(크기 차수)
3. **가수 (Mantissa / Significand)** — **유효 숫자** 쪽 정밀도

### 정규화와 Hidden Bit

실수를 **1.xxx × 2ⁿ** 꼴로 맞추는 과정이 **정규화**입니다. 이때 소수점 앞은 항상 **1**이라서, 메모리에는 안 넣고 생략하기도 하는데 이를 **Hidden Bit**라고 합니다. 가수 비트를 아끼면서 정밀도를 올리는 트릭입니다.

### 초과 표현 (Excess / Bias)

지수에 **음수**도 필요하지만, 지수만 2의 보수로 넣으면 **실수끼리 비교**가 번거로워질 수 있습니다. 그래서 **Bias를 더해 모두 양수처럼** 저장해 두고, 하드웨어에서는 **정수 비교처럼** 빠르게 순서를 매깁니다.

## 질의 응답

### 1. 2의 보수와 1의 보수: '0'이 두 개인 이유와 한계

“실제 2진수에서는 0이 하나로 표현된다”라고 알고 계신 것이 **정확한 현대 컴퓨터의 작동 원리**입니다. 교재에서 설명하는 내용은 1의 보수 방식이 초창기에 가졌던 ‘치명적인 결함’을 설명하기 위함입니다.

#### 보수를 왜 쓰나

**한 줄 요약:** 음수를 비트에 **정해진 규칙(보수)**으로 넣어 두면, 하드웨어가 **`a − b`를 `a + (b의 음수 표현)`**처럼 바꿔서 **뺄셈 전용 회로 없이 덧셈기만**으로 처리할 수 있습니다.

**비유:** 식당 계산대에 **덧셈만 되는 계산기**가 하나 있다고 칩니다. 손님이 “5000원짜리에서 2000원 빼 주세요”라고 하면, 직원이 **빼기 버튼**이 없으면 난감합니다. 그래서 “빼야 할 2000원”을 **‘더해서 0이 되게 만드는 수’**로 바꿔 적어 두면, 결국 **한 종류의 연산(더하기)**만 반복해도 됩니다. CPU 입장에서 **보수**가 그 “빼기를 더하기로 바꿔 쓰는 약속”입니다.

**작은 예:** **일의 자리만** 본다고 치면 `5 − 3`을 `5 + 7 = 12`에서 **맨 오른쪽 한 자리만** 읽으면 `2`가 됩니다. 여기서 7은 **10을 기준으로 3을 상쇄하는 쪽의 수**에 가깝습니다. 이진 2의 보수도 **“비트 n개짜리 상자 안에서 음수를 그런 식으로 잡자”**는 약속입니다.

#### 2의 보수를 쉽게 잡기

**비트 개수가 정해져 있다**는 전제가 핵심입니다. 예를 들어 4비트면 `0000`부터 `1111`까지만 쓸 수 있고, 여기서 **맨 앞 비트를 부호**처럼 쓰기로 약속하면 “같은 비트열”이 양수일 수도, 음수일 수도 있게 됩니다.

- **직관:** 시계에서 11시에서 **2시간 뒤**를 생각해 보면, 앞으로 두 칸 가도 되고 **뒤로 열 칸** 가도 1시가 됩니다. 고정된 칸(비트 수) 안에서 **“빼는 것”을 “큰 수를 더하는 것”으로 바꿔 쓰는 트릭**이 2의 보수에 가깝습니다. 그래서 CPU는 **뺄셈 전용 회로 없이 덧셈기만**으로 가감을 처리할 수 있습니다.

**음수(2의 보수) 만드는 법**은 보통 두 단계로 외웁니다.

1. 양수 비트를 **전부 뒤집기** (0↔1) → 이게 **1의 보수**에 해당하는 모양입니다.
2. 거기에 **1을 더하기** → 이게 **2의 보수** 음수 표현입니다.

**4비트 예 (3과 −3):**

이제 **`0011 + 1101`**을 그냥 덧셈하면 자리 올림으로 `10000`이 되는데, **4비트만 남기면** `0000`이 됩니다. 즉 **3 + (−3) = 0**이 덧셈 한 번으로 맞아 떨어집니다.

**0이 하나만 남는 이유(2의 보수 쪽):** 1의 보수에서는 `0000`(+0)을 뒤집으면 `1111`(−0)이라 **0이 두 얼굴**이 됩니다. 2의 보수 규칙으로 음수를 정하면 −0에 해당하는 비트열이 사라지고, **0은 `0000…` 하나**로 정리됩니다.

#### 왜 1의 보수를 안 쓰고 2의 보수를 쓰나

**비유:** 엘리베이터 층수 표시에 **“0층”**과 **“지하 0층”**이라는 말이 따로 붙어 있고, 둘 다 “지상과 같은 높이”라면, 안내 방송과 프로그램이 **둘 다 0으로 처리해야 하는지** 매번 따져야 합니다. 1의 보수의 **`+0` / `−0`**은 값은 같은데 **비트 패턴만 다른** 그런 상황에 가깝습니다. 비교·분기·최적화 같은 걸 할 때 **규칙이 지저분**해집니다.

**덧셈 예시(4비트, 직관만):** 1의 보수로 `1 + (−1)`을 하면 `0001 + 1110 = 1111`인데, 이 비트열은 **`−0`**에 해당합니다. **진짜 0(`0000`)**이 되려면 **끝에서 생긴 올림을 맨 앞으로 돌려서 한 번 더 더하는** 식의 **추가 보정**이 필요한 경우가 있습니다. 반면 2의 보수에서는 `1`과 `−1`을 **`0001 + 1111`**로 더하면 `10000`이 되고, **4비트만 쓰면 `0000`**으로 떨어져 **평범한 덧셈 한 번**으로 끝나기 쉽습니다.

**정리:** “보수를 쓰자”는 쪽은 1의 보수도 만족하지만, **0을 하나로** 맞추고 **덧셈·뺄셈 처리를 단순하게** 가져가려고 실무·교재 표준은 **2의 보수**로 통일된 경우가 많습니다.

- **이론적 배경:** 1의 보수 방식은 이진수의 0을 1로, 1을 0으로 반전시켜 음수를 표현합니다. 이 규칙을 적용하면 +0은 `0000`이 되고, 이를 반전시킨 −0은 `1111`이 됩니다. 즉, 수학적으로 완전히 동일한 0이라는 값에 두 가지 이진수 표현이 생기는 모순이 발생합니다.
- **결론:** 현실 세계의 수학에서 +0과 −0이 구분되지 않듯, 컴퓨터 시스템 내부에 0이 두 개 존재하면 연산에 심각한 논리적 오류를 유발합니다. 따라서 현대 컴퓨터는 이 문제를 해결하기 위해, 1의 보수에 1을 더하여 0을 하나(`0000`)로 통일시키는 **2의 보수 표현**을 범용적인 표준으로 사용하고 있습니다.

### 3. BCD 입력과 7-세그먼트 출력의 작동 원리(TMI)

7-세그먼트는 디지털 시계나 엘리베이터 층수 표시기에서 흔히 볼 수 있는 '8'자 모양의 디스플레이입니다. 이 디스플레이는 총 7개의 LED 막대 조각(a부터 g까지 지정)으로 구성되어 있습니다.

- **이론과 비유:** BCD(Binary Coded Decimal)는 0부터 9까지의 십진수를 4비트 이진수로 표현한 '입력 신호'입니다. 이 신호가 조합 논리 회로에 전달되면, 회로는 설계된 규칙에 따라 어떤 LED를 켤지 판단하는 '두뇌' 역할을 수행합니다.
- **작동 예시:** 사용자가 숫자 ‘8’에 해당하는 BCD 값(`1000`)을 입력하면, 회로는 디스플레이를 숫자 8의 형태로 만들기 위해 7개의 모든 LED 조각(a~g)에 출력 신호 ‘1’을 보내 불을 켭니다. 반대로 숫자 ‘1’(`0001`)을 입력하면, 숫자 1의 형태를 완성하기 위해 우측의 세로 막대인 b와 c 조각에만 불을 켜도록 제어합니다. 이것이 BCD 입력값에 따라 a~g 출력이 달라지는 원리입니다.

Don't Care 조건(X)은 시스템의 특성상 '절대로 입력될 일이 없는 경우의 수'를 의미합니다 -> 일단 묶고 안 쓰는 애들은 버린다. 예를 들어 BCD 코드는 0부터 9까지만 사용하므로, 10부터 15까지의 값은 입력될 일이 없어 X로 처리됩니다.

- **비유:** 카르노 맵에서 'X'는 카드 게임의 조커(Joker)와 같습니다. 식을 가장 간단하게 만들기 위해 필요하다면 '1'로 취급하여 다른 1들과 크게 묶어도 되고, 필요 없다면 '0'으로 간주하여 무시해도 됩니다. K-map의 핵심 목표는 '1'들을 최대한 크게 묶어 논리 게이트를 최소화하는 것입니다.
- **계산 과정:** 질문하신 AB=10, CD=00 (반드시 켜야 하는 1)과 AB=10, CD=10 (조커로 활용할 X)의 두 칸을 하나의 그룹으로 묶었을 때의 계산은 다음과 같습니다. 묶인 그룹 내에서 변하지 않는 공통 변수만 추출합니다.
  - A는 두 칸 모두 1이므로 A
  - B는 두 칸 모두 0이므로 B′
  - C는 0에서 1로 상태가 변했으므로 식에서 소거합니다.
  - D는 두 칸 모두 0이므로 D′
- **결론:** 변화가 발생한 C가 소거되면서, 최종적인 최소화 수식은 **AB′D′**가 됩니다. Don't Care(X)를 ‘1’로 활용해 묶음의 크기를 키움으로써, 회로를 구성하는 논리 변수를 하나 줄이는 최적화에 성공한 것입니다.

#### 적용 예시

**예시 1 — `A′ + A·B` 줄이기**  
**OR 분배** `X + Y·Z = (X+Y)·(X+Z)` 로 묶으면 `(A′+A)·(A′+B)` 이고, 여기서 **`A′+A = 1`** 이므로 **`A′ + B`** 까지 됩니다.

---

**예시 2 — `A·B + A·B′` 줄이기**  
앞에 **`A` 공통**으로 빼면 `A·(B + B′)` 이고, **`B + B′ = 1`** 이라서 **`A`** 만 남습니다.

---

**예시 3 — `A + A·B` 줄이기**  
**`A` 공통**으로 `A·(1 + B)` 로 쓰고, **`1 + B = 1`** (OR에서 1이면 전체 1) 이므로 **`A`** 입니다.

---

#### 정리 표 (한눈에)

- **항등:** A+0=A, A·1=A
- **영:** A+1=1, A·0=0
- **멱등:** A+A=A, A·A=A
- **보수:** A+A′=1, A·A′=0, (A′)′=A
- **교환·결합·분배:** 위 표 참고
- **흡수:** A+AB=A, A(A+B)=A
- **드모르간:** (AB)′=A′+B′, (A+B)′=A′B′

K-map·논리식 최소화 문제에서 **흡수·드모르간·분배**를 많이 씁니다.

### 7. 논리도 ↔ 진리표 (시험·과제에서 맞는 방향)

7번에서는 **논리도를 보고 진리표를 채우는 것**과, **진리표를 보고 논리도를 그리는 것**을 **둘 다** 연습합니다. 방향만 바뀌었을 뿐, 쓰는 도구는 위 **게이트 기호**와 **AND / OR / NOT** 의미가 같습니다.

#### A. 논리도 → 진리표

<Image src="/cs_week1/gate_2.png" alt="논리게이트" width={800} height={500} />

**도 → 표** 연습은 **직접 만든 논리도**나 **교재·과제지 그림**을 두고, 아래 순서대로 **2ⁿ행 진리표**를 채우면 됩니다.

1. **입력 변수 개수**를 센 뒤, 진리표 **앞쪽 열**에 가능한 조합을 모두 씁니다. 입력이 **n개**이면 **2ⁿ행**(2를 n번 곱한 개수)입니다.
2. **한 행씩** 고정합니다 (예: `A=0, B=1`). 그 조합을 도의 **입력 쪽**에 넣었다고 생각하고, **게이트를 지나는 순서대로** 중간 결과를 적어 나갑니다.
3. **NOT**은 들어온 값을 뒤집고, **AND**는 모두 1일 때만 1, **OR**는 하나라도 1이면 1로 계산합니다.
4. 마지막 **출력 단자**에 나온 값이 그 행의 출력 열입니다. 모든 행을 채우면 끝입니다.

**팁:** 가지가 갈리면 **같은 입력이 두 갈래로** 들어가도 각각 **같은 0/1**로 읽으면 됩니다. 중간에 적어 둔 값이 틀리면 그 다음 게이트가 연쇄로 틀어지니, **한 행 안에서는 앞에서부터 차근차근** 확인하는 습관이 좋습니다.

#### B. 진리표 → 논리도

1. **출력이 1인 행**만 골라서, 그때 **입력들의 AND 조합**을 만듭니다 (각 변수는 그 행에서 0이면 `NOT`을 붙인 형태, 1이면 그대로).
2. 그렇게 만든 **항들을 OR로 이어 붙이면** 됩니다. (교재 말로는 **표준 합의곱형**, **minterm 합** 쪽에 해당합니다.)
3. 도로 옮길 때는 **각 minterm**을 **AND 게이트 묶음**(필요하면 입력에 NOT)으로 그리고, 여러 줄을 **OR 게이트**로 모으면 논리도가 됩니다.
4. 문제에서 “최소 게이트”·“식 간단히”를 요구하면, 먼저 **불대수나 K-map으로 식을 줄인 뒤** 그 식에 맞춰 다시 그리면 됩니다. **7번 기본형**은 “진리표 그대로 → SOP 한 번 올리기”만 해도 됩니다.

**아주 작은 예:** 입력 `A, B`에 대해 출력 `Y`가 **`A=1` 이고 `B=1`일 때만 1**이면, 진리표에서 1인 행은 한 줄뿐이므로 **`Y = A · B`** → **AND 게이트 하나**면 됩니다.

#### 체크리스트

- **도 → 표:** 2ⁿ행 빠짐없이 / 한 행씩 중간값까지 추적했는지
- **표 → 도:** 출력 1인 행마다 AND 한 덩어리 → 전체 OR / NOT 위치(0인 변수) 빠뜨리지 않았는지
