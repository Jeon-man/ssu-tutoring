이번 주차에서는 컴퓨터가 실수를 메모리에 저장하고 처리하는 방식을 다룬다. 특히 부동 소수점의 원리와 한계를 이해하는 것이 핵심이다.

## 1. 실수의 개요와 C 언어의 자료형

컴퓨터 내부에서 실수를 표현하는 방식은 정수와 근본적으로 다르다. C 언어에서 사용하는 대표적인 실수 자료형은 다음과 같다.

### C 언어의 실수 자료형

- **float:** 크기는 4바이트이며, 표현할 수 있는 값의 범위는 대략 -3.4 x 10^-38 에서 3.4 x 10^38 까지이다.

- **double:** 크기는 8바이트이며, 표현할 수 있는 값의 범위는 대략 -1.7 x 10^-308 에서 1.7 x 10^308 까지이다.

`double`형은 `float`형보다 메모리를 두 배 더 사용하지만, 훨씬 넓은 범위의 수를 더 정밀하게 표현할 수 있다.

---

## 2. 실수의 정밀도와 오차 문제

컴퓨터는 유한한 비트로 무한한 실수를 표현해야 하므로 필연적으로 근사치를 사용하게 되며, 이 과정에서 **오차**가 발생한다.

### 실습 예제: 실수 연산의 한계

```c
#include <stdio.h>

void float_test() {
    float a = 0.2;
    float b = 0.2;

    if (a + b == 0.4) {
        printf("정확함\n");
    } else {
        printf("오차 발생\n"); // 실제 실행되는 구간
    }
}
```

- **원인:** 코드 상의 상수 `0.4`는 기본적으로 `double` 타입으로 처리된다. 반면 `a + b`는 `float` 타입의 연산 결과다. 정밀도가 다른 두 값을 비교할 때 미세한 오차로 인해 두 값이 다르다고 판별된다.

- **주의점:** 컴퓨터에서 실수를 다룰 때, 덧셈을 반복하면 오차가 누적된다. 따라서 정밀도가 중요한 계산에서는 반복적인 덧셈보다 한 번의 곱셈 연산을 사용하는 것이 오차를 줄이는 방법이다.

---

## 3. 실수 표현 방식: 고정 소수점과 부동 소수점

- **고정 소수점 (Fixed-Point):** 소수점의 위치를 미리 고정해 두는 방식이다. (예: 16비트 중 상위 8비트는 정수부, 하위 8비트는 소수부). 연산 속도는 빠르지만, 표현할 수 있는 수의 범위가 매우 제한적이다.

- **부동 소수점 (Floating-Point):** 소수점의 위치가 이동할 수 있는 방식이다. 과학적 표기법($a \times 10^b$)을 2진수에 적용한 형태로, 현재 대부분의 시스템이 채택하고 있는 **IEEE 754** 표준 규격이다.

---

## 4. 부동 소수점의 구조 (IEEE 754)

부동 소수점은 크게 세 가지 영역으로 나뉜다.

1. **부호 (Sign):** 양수는 `0`, 음수는 `1`로 표현한다.
2. **지수 (Exponent):** 숫자의 전체적인 크기(범위)를 결정한다.
3. **가수 (Mantissa / Significand):** 숫자의 정밀도를 결정하는 유효 숫자 부분이다.

### 핵심 개념 1: 정규화 (Normalization)와 Hidden Bit

모든 2진수 실수를 1.xxx 2^n 형태로 변환하는 과정을 정규화라고 한다. 정규화를 거치면 소수점 앞의 자리는 항상 `1`이 된다. 항상 고정된 값이므로 메모리에 굳이 저장하지 않고 생략하는데, 이를 **Hidden Bit**라고 부른다. 이를 통해 가수 부분에 1비트의 여유 공간을 확보하여 정밀도를 높일 수 있다.

### 핵심 개념 2: 초과 표현 (Excess / Bias)

지수에는 음수가 포함될 수 있다. 그러나 지수 영역에 음수를 2의 보수 방식으로 저장하면, 하드웨어(CPU)가 두 실수의 대소를 비교할 때 연산이 복잡해진다.
이 문제를 해결하기 위해 지수에 일정한 값(Bias)을 더하여 모두 양수로 변환하여 저장한다. 이를 통해 실수 간의 대소 비교를 단순한 정수 비교처럼 빠르게 처리할 수 있다.

---

## 5. 특수 값 (Special Values)

- **0 (Zero):** 모든 비트가 0인 상태.
- **Overflow:** 표현 가능한 최대값을 초과한 상태.
- **Underflow:** 0에 너무 가까워져서 표현 가능한 최소값보다 작아진 상태.
- **NaN (Not a Number):** 0을 0으로 나누는 등 수학적으로 정의되지 않은 연산의 결과.
