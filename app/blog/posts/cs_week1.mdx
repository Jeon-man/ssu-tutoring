## 2. 실수의 정밀도와 오차 문제

비트 수는 유한한데 실수는 무한에 가깝기 때문에 **근사값**을 쓰게 되고, 그 과정에서 **오차**가 생깁니다.

### 실습 예제: 실수 연산의 한계

```c
#include <stdio.h>

void float_test() {
    float a = 0.2;
    float b = 0.2;

    if (a + b == 0.4) {
        printf("정확함\n");
    } else {
        printf("오차 발생\n"); // 실제 실행되는 구간
    }
}
```

**왜 이런 일이 나는가**

- 상수 `0.4`는 보통 **`double`**으로 해석되고, `a + b`는 **`float`** 연산 결과입니다. **비트 수·해석 방식이 다른 두 값**을 `==`로 비교하면 미세한 차이 때문에 다르다고 나올 수 있습니다.

**실무에서의 습관**

- 정밀도가 중요하면 **반복 덧셈**보다 **한 번의 곱셈** 등으로 오차 누적을 줄이는 방향을 고려합니다.

## 4. 부동 소수점의 구조 (IEEE 754)

한 실수를 크게 **세 덩어리**로 나눕니다.

1. **부호 (Sign)** — 양수 `0`, 음수 `1`
2. **지수 (Exponent)** — 전체 **스케일**(크기 차수)
3. **가수 (Mantissa / Significand)** — **유효 숫자** 쪽 정밀도

### 정규화와 Hidden Bit

실수를 **1.xxx × 2ⁿ** 꼴로 맞추는 과정이 **정규화**입니다. 이때 소수점 앞은 항상 **1**이라서, 메모리에는 안 넣고 생략하기도 하는데 이를 **Hidden Bit**라고 합니다. 가수 비트를 아끼면서 정밀도를 올리는 트릭입니다.

### 초과 표현 (Excess / Bias)

지수에 **음수**도 필요하지만, 지수만 2의 보수로 넣으면 **실수끼리 비교**가 번거로워질 수 있습니다. 그래서 **Bias를 더해 모두 양수처럼** 저장해 두고, 하드웨어에서는 **정수 비교처럼** 빠르게 순서를 매깁니다.

## 질의 응답

### 1. 2의 보수와 1의 보수: '0'이 두 개인 이유와 한계

“실제 2진수에서는 0이 하나로 표현된다”라고 알고 계신 것이 **정확한 현대 컴퓨터의 작동 원리**입니다. 교재에서 설명하는 내용은 1의 보수 방식이 초창기에 가졌던 ‘치명적인 결함’을 설명하기 위함입니다.

- **이론적 배경:** 1의 보수 방식은 이진수의 0을 1로, 1을 0으로 반전시켜 음수를 표현합니다. 이 규칙을 적용하면 +0은 `0000`이 되고, 이를 반전시킨 −0은 `1111`이 됩니다. 즉, 수학적으로 완전히 동일한 0이라는 값에 두 가지 이진수 표현이 생기는 모순이 발생합니다.
- **결론:** 현실 세계의 수학에서 +0과 −0이 구분되지 않듯, 컴퓨터 시스템 내부에 0이 두 개 존재하면 연산에 심각한 논리적 오류를 유발합니다. 따라서 현대 컴퓨터는 이 문제를 해결하기 위해, 1의 보수에 1을 더하여 0을 하나(`0000`)로 통일시키는 **2의 보수 표현**을 범용적인 표준으로 사용하고 있습니다.

### 3. BCD 입력과 7-세그먼트 출력의 작동 원리(TMI)

7-세그먼트는 디지털 시계나 엘리베이터 층수 표시기에서 흔히 볼 수 있는 '8'자 모양의 디스플레이입니다. 이 디스플레이는 총 7개의 LED 막대 조각(a부터 g까지 지정)으로 구성되어 있습니다.

- **이론과 비유:** BCD(Binary Coded Decimal)는 0부터 9까지의 십진수를 4비트 이진수로 표현한 '입력 신호'입니다. 이 신호가 조합 논리 회로에 전달되면, 회로는 설계된 규칙에 따라 어떤 LED를 켤지 판단하는 '두뇌' 역할을 수행합니다.
- **작동 예시:** 사용자가 숫자 ‘8’에 해당하는 BCD 값(`1000`)을 입력하면, 회로는 디스플레이를 숫자 8의 형태로 만들기 위해 7개의 모든 LED 조각(a~g)에 출력 신호 ‘1’을 보내 불을 켭니다. 반대로 숫자 ‘1’(`0001`)을 입력하면, 숫자 1의 형태를 완성하기 위해 우측의 세로 막대인 b와 c 조각에만 불을 켜도록 제어합니다. 이것이 BCD 입력값에 따라 a~g 출력이 달라지는 원리입니다.

Don't Care 조건(X)은 시스템의 특성상 '절대로 입력될 일이 없는 경우의 수'를 의미합니다 -> 일단 묶고 안 쓰는 애들은 버린다. 예를 들어 BCD 코드는 0부터 9까지만 사용하므로, 10부터 15까지의 값은 입력될 일이 없어 X로 처리됩니다.

- **비유:** 카르노 맵에서 'X'는 카드 게임의 조커(Joker)와 같습니다. 식을 가장 간단하게 만들기 위해 필요하다면 '1'로 취급하여 다른 1들과 크게 묶어도 되고, 필요 없다면 '0'으로 간주하여 무시해도 됩니다. K-map의 핵심 목표는 '1'들을 최대한 크게 묶어 논리 게이트를 최소화하는 것입니다.
- **계산 과정:** 질문하신 AB=10, CD=00 (반드시 켜야 하는 1)과 AB=10, CD=10 (조커로 활용할 X)의 두 칸을 하나의 그룹으로 묶었을 때의 계산은 다음과 같습니다. 묶인 그룹 내에서 변하지 않는 공통 변수만 추출합니다.
  - A는 두 칸 모두 1이므로 A
  - B는 두 칸 모두 0이므로 B′
  - C는 0에서 1로 상태가 변했으므로 식에서 소거합니다.
  - D는 두 칸 모두 0이므로 D′
- **결론:** 변화가 발생한 C가 소거되면서, 최종적인 최소화 수식은 **AB′D′**가 됩니다. Don't Care(X)를 ‘1’로 활용해 묶음의 크기를 키움으로써, 회로를 구성하는 논리 변수를 하나 줄이는 최적화에 성공한 것입니다.
